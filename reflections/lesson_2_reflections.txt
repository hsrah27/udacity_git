What happens when you initialize a repository? Why do you need to do it?
git init turns a folder into a git repository which means that you can start tracking files and changes to those files

How is the staging area different from the working directory and the repository? What value do you think it offers?
The staging area is an area where you can stage files to be committed; this allows you to group some files, out of
all of the changed files in the working directory, so that the grouped files can be committed together

How can you use the staging area to make sure you have one commit per logical change?
add files to the staging area as you make changes ... compare your working directory to your staging area with git diff
... compare your staging area to the most recent commit in your repository with git diff --staged ...
commit your changes from your staging area to your repository

What are some situations when branches would be helpful in keeping your history organized? How would branches help?
If you want to experiment with a new feature, you can use a new branch. Or if you want to create a new version of a
project, you can create a new branch
